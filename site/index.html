技术细节

原生、轻量、可维护

2.0 把六年前的代码全部推倒重写，使用最新的 Manifest V3 规范、原生 DOM、ES Modules。

  • Manifest V3 服务工作者：用 chrome.alarms 替代后台长驻 setInterval。
  • 原生 API 优先：MutationObserver、IntersectionObserver、navigator.clipboard，告别 jQuery。
  • CSP 友好的计算器：自实现的递归下降解析器，无 eval、new Function。
  • 最小权限：移除 webRequestBlocking，仅保留必要的 contextMenus / storage / alarms / activeTab。
  • 每个功能一个文件：站点增强放在 src/content/sites/，新加站点不会影响其他逻辑。
manifest.json 
{
  "manifest_version": 3,
  "name": "集盒 SetBox",
  "version": "2.1.0",
  "background": {
    "service_worker": "src/background/service-worker.js",
    "type": "module"
  },
  "permissions": [
    "contextMenus", "storage",
    "activeTab", "tabs", "alarms"
  ]
}

常见问题

FAQ

如何安装？

1. 从 下载页面 获取压缩包并解压。

2. 浏览器地址栏输入 chrome://extensions（Edge 用 edge://extensions）。

3. 打开右上角「开发者模式」，点击「加载已解压的扩展」选择解压后的目录即可。

支持哪些浏览器？

所有基于 Chromium 的浏览器都可使用，包括 Chrome、Edge、Brave、Arc、Vivaldi 等。Manifest V3 规范，未来 Firefox 也将兼容。

翻译会上传我的数据吗？

划词翻译只把你点击翻译的那段文字发送到 Bing / Google 翻译公开接口，扩展自身不持有任何服务器，不收集、不记录、不外传。源代码全开，可自行审计。

定时刷新最小间隔为什么是 30 秒？

这是 Chrome 对 chrome.alarms 的硬性限制，并非扩展所能控制。换来的好处是即便浏览器进入后台、Service Worker 被回收，定时器仍然可靠。

如何反馈问题或贡献代码？

在 GitHub Issues 提交问题或建议，欢迎 Pull Request。每个站点增强是单独文件，新增站点门槛极低。
